\section{Manim}
The Manim library\cite{manim} was originally developed by Grant Sanderson\cite{3b1b-website} for his educational YouTube channel ``3Blue1Brown''\cite{3b1b-channel} to create animations of mathematical and computer-science concepts. Since its release, the channel has accumulated over 6~million subscribers, demonstrating the educational impact of smooth, precisely choreographed animations (Fig.~\ref{fig:manim_example}) that build intuition about abstract concepts. Given this history, Manim represented an~excellent candidate for creating high-quality educational visualizations of program behavior.

Manim's strength lies in its ability to programmatically generate animations with mathematical precision. Instead of tedious keyframe animation, users write Python code to determine how objects should appear, change, and interact. Manim can render 3D scenes, typeset \LaTeX{} equations, and output crisp 60\,fps animations suitable for anything from YouTube tutorials to academic conferences. For education, its capacity to make complicated ideas---like matrix transformations, function composition, or program flow---tangible and understandable by systematically unfolding them is particularly valuable.

Yet this power comes with a barrier to entry. A professor preparing a single lecture, or a researcher creating visualizations for a conference talk, may have a clear idea of what they want to show---perhaps how a recursive function builds up its stack---but writing Manim code for it requires a distinct skill set. LLVManim addressed this challenge by automatically translating program semantics into Manim scenes, thereby eliminating the barrier of manual coding between LLVM's analytical power and Manim's visual sophistication.
